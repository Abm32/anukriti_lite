Anukriti is an **In Silico Pharmacogenomics Platform** (Version 0.4 Beta) that simulates drug effects on synthetic patient cohorts using Agentic AI. The system predicts how patients will respond to medications based on their genetic profile, medical history, and lifestyle factors with comprehensive multi-enzyme coverage and CPIC guideline compliance.

## Core Functionality

- **Molecular Analysis**: Converts drug SMILES strings to 2048-bit Morgan fingerprints using RDKit with structural analysis capabilities and enhanced validation
- **Similarity Search**: Finds similar drugs using vector similarity search (Pinecone or OpenSearch when configured; falls back to mock data when no vector backend credentials are present). Enhanced metadata includes SMILES, targets, and side effects.
- **Genetic Profiling**: Processes VCF files to extract pharmacogene variants across 8 chromosomes (2, 6, 10, 11, 12, 16, 19, 22) covering CYP2D6, CYP2C19, CYP2C9, UGT1A1, SLCO1B1, VKORC1. **Database Backend Operational & Integrated (Week 1 Complete)**: SQLite database with 39 genes operational (15 Tier 1 + 16 Tier 2 + 8 Tier 3), expanding to 40 genes (Week 2), 100+ genes (Month 3), 200+ genes (Month 12). Automated PharmVar/CPIC synchronization enables 24-48x faster gene addition (5 minutes vs 2-4 hours). **Direct S3/HTTPS Access**: Supports streaming from 1000 Genomes AWS Public Dataset without downloading files (zero storage cost, tabix HTTP range requests).
- **Advanced CNV Detection (PLANNED - Month 1)**: CYP2D6 copy number variation detection (0-6 copies) using digital PCR-inspired algorithms for structural variant resolution, addressing competition feedback on incomplete CNV detection.
- **Clinical Validation Framework (IMPLEMENTED - Week 1 Day 2)**: Coriell reference sample validation test suite (10 samples, expandable to 50+, 95% concordance target), PharmCAT comparison framework (100 diverse samples, 90-95% concordance target), CPIC compliance audit. Comprehensive validation documentation (6,800 words) ready for regulatory submission. Academic partnerships for retrospective clinical studies (Month 1). Peer-reviewed publication pipeline established (Month 3).
- **FDA Regulatory Compliance (NEW - Week 1)**: Non-Device CDS qualification under 21st Century Cures Act documented. Clear regulatory pathway: QMS implementation (Month 2) → FDA Pre-Submission (Month 3) → Breakthrough Device application (Month 6) → International expansion (Months 6-12).
- **Multi-Backend LLM Resilience (IMPLEMENTED - Week 1 Complete)**: Automatic failover across Nova → Bedrock Claude → Gemini → Anthropic → Deterministic fallback. Rate limiting, caching, and multi-region deployment ensure 99.9% uptime. Pre-computed demo scenarios for critical use cases.
- **Targeted Variant Lookup**: Database-backed genotyping using Tier 1 Clinical Variants (CPIC Level A) from PharmVar database, replacing naive variant counting with accurate phenotype prediction. **Production Enhancement (Operational - Week 1 Complete)**: SQLite database backend (`pharmacogenes.db`) enables dynamic gene loading and sub-100ms query performance for 39 genes (15 Tier 1 + 16 Tier 2 + 8 Tier 3), expandable to 100+ genes.
- **Expanded Pharmacogene Panel**: Analyzes 8 chromosomes for comprehensive pharmacogenomic profiling with integrity validation. **Database Foundation Complete (Week 1 Complete)**: 39 genes operational in database (15 Tier 1 + 16 Tier 2 + 8 Tier 3) with automated PharmVar/CPIC synchronization (24-48x faster gene addition: 5 minutes vs 2-4 hours), expandable to 100+ genes.
- **Deterministic PGx Engine**: CPIC/PharmVar guideline-based allele calling with database-backed variant storage (39 genes operational: 15 Tier 1 + 16 Tier 2 + 8 Tier 3) - no LLM in decision layer. **Database Backend (Implemented & Integrated - Week 1 Complete)**: Replaces hardcoded dictionaries with scalable SQLite backend, enabling 100+ gene support with automated data pipeline (24-48x faster: 5 minutes vs 2-4 hours per gene). Integrated with `allele_caller.py` and `vcf_processor.py` (Day 1 afternoon). PharmVar/CPIC sync complete (Day 2). Multi-backend LLM resilience complete (Days 3-7).
- **Drug-triggered PGx**: Context-aware gene display showing only drug-relevant pharmacogenes (Warfarin → CYP2C9 + VKORC1; Statins → SLCO1B1; Clopidogrel → CYP2C19; Thiopurines → TPMT; Fluoropyrimidines → DPYD; Abacavir → HLA_B5701; Irinotecan → UGT1A1)
- **Local CPIC Retrieval**: Versioned PGx data in repo (`data/pgx/`) for reproducible, offline allele calling - no runtime API dependencies
- **AI Simulation**: Default LLM is **Amazon Nova Lite/Pro on AWS Bedrock** (`LLM_BACKEND=nova`). Also supports Google Gemini (`LLM_BACKEND=gemini`), Bedrock Claude (`LLM_BACKEND=bedrock`), and direct Anthropic Claude API (`LLM_BACKEND=claude` via `langchain-anthropic`, requires `ANTHROPIC_API_KEY`). Uses hybrid RAG (Titan embeddings over local CPIC JSON + molecular fingerprint similarity over ChEMBL) and CPIC guideline-based prompting to predict drug responses. Supports per-request backend selection. **Key Innovation**: Hybrid architecture where deterministic CPIC/PharmVar engine makes decisions and LLM provides natural language explanation — differentiating from "GPT wrapper" projects where LLM makes clinical decisions.
- **Novel Drug Inference**: `src/novel_drug_inference.py` infers candidate pharmacogenes for novel compounds via analog retrieval and metadata. `src/confidence_tiering.py` classifies outputs as `high`, `moderate`, or `exploratory`. Exposed via `POST /analyze/novel-drug` with explicit `decision_grade` and validation artifact.
